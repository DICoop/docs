# Платформа

Платформа "Цифровой Экономики" - системообразующее программное обеспечение для удовлетворения потребностей. Платформа предоставляет высокотехнологичную программную среду для автоматизации деятельности потребительских и производственных кооперативов на основе методологии кооперации Союза Потребительских Обществ “РУСЬ”, которая позволяет:

- Удаленно принимать пайщиков в кооперативы;

- Вести Реестр пайщиков в соответствии с нормативными актами России;

- Проводить дистанционные голосования пайщиков и членов Совета с автоматическим созданием Протоколов голосования;

- Осуществлять учет вступительных, членских и паевых взносов, а также возврата паевых взносов;

- Осуществлять учет расхода потребительским кооперативом членских взносов пайщиков;

- Вести автоматизированный первичный документооборот по типовым целевым потребительским программам;

Все юридические действия пайщиков, членов Совета, Председателя кооператива регистрируются в системе распределенного реестра, на основе которого формируется документы, подписанные простой электронной подписью в режиме реального времени: 

- Заявления; 

- Протоколы голосования; 

- Решения Совета; 

- Акты приема-передачи;

Кооперативы в цифровой среде кооперативной экономики обладают возможностью бесшовной, интегративной кооперации, когда пайщики одного кооператива получают доступ к продуктам другого только на основании внутренних цифровых документов без их выхода их из общей цифровой среды. 

Платформа создана для кооперативов и удовлетворения ими потребностей пайщиков, в том числе, в длинных цепочках создания ценности и планирования её производства. Ниже представлено краткое, поверхностное описание Платформы, позволяющее создать общее представление о ней. 


## Организационная архитектура
### Оператор платформы
Оператором платформы является потребительский кооператив "ВОСХОД", который осуществляет подключение кооперативов к цифровой среде. После подключения, кооператив получает возможность использовать среду для электронного документооборота. 


### Провайдеры платформы
Провайдером программного обеспечения, реализующим спецификацию платформы, может стать любой подключенный кооператив, и тем самым, поставлять свое программное обеспечение в экосистему. Программное обеспечение становится доступным для использования всеми кооперативами на условиях провайдера.  

Одним из провайдеров является ПК ВОСХОД, предоставляющий продукт "Цифровой Кооператив", реализующий систему персональных рабочих столов для пайщиков. 


### Делегаты платформы
Делегаты кооперативной экономики - это 21 основная команда и 150 резервных команд людей, обеспечивающих исполнение программного комплекса кооперативной экономики на своих серверах. 

Делегаты поставляют вычислительные ресурсы своих серверов - кооперативам, которые используются ими в своей деятельности. Вычислительные ресурсы состоят из:
- Оперативной памяти, которую используют кооперативы для хранения информации;
- Время вычислений процессора, которое задействуют кооперативы;
- Объем информации, который передают кооперативы;

Таким образом, кооперативы, которые подключаются к платформе, используют вычислительные ресурсы серверов делегатов, за что делегаты получают вознаграждение.

### Клиенты платформы
Клиентами являются кооперативы, осуществляющие автоматизацию своей деятельности посредством программного обеспечения Провайдеров на серверах Делегатов платформы Кооперативной Экономики. 



## Программная архитектура

### Ядро
Основанием платформы является операционная блокчейн-система, реализующая программную среду для системных и стандартизированных смарт-контрактов цифровых целевых потребительских программ. 

Операционная система содержит следующие низкороувневые программные протоколы, обеспечивающие работу:

- Протокол Программного Интерфейса (API) - предоставляет конечные точки для программного взаимодействия с Платформой;

- Протокол Сети (NET) - обеспечивает внутрисистемную коммуникацию по передаче информации между Делегатами Платформы;

- Протокол Консенсуса (CONSUNSUS) - обеспечивает низкоровневые правила по определению состояния распределенной базы данных;

- Протокол Состояния (STATE) - обеспечивает формирование распределенной базы данных;

- Протокол Виртуальной Машины (WASM) - обеспечивает исполнение смарт-контрактов в распределенной базе данных;

- Протокол Истории (HISTORY) - обеспечивает формирование истории всех действий на платформе;

- Протокол Контроля Доступа (AUTH) - обеспечивает проверку контроля прав доступа при совершении действий на платформе;

И программные продукты, реализующие работу этих протоколов:

- nodeos - 
- cleos - 
- keosd - 

Все эти протоколы и программные продукты являются комплексным программным обеспечением ядра платформа кооперативной экономики, написанным на языке программирования C/C++, и обеспечивают работу системы на самом низком (техническом) уровне.

### Смарт-контракты

Смарт-контракты реализуют стандартизированные целевые потребительские программы кооперативов. Смарт-контракты исполняются на серверах Делегатов, в их виртуальных машинах, которые связаны между собой по протоколам NET, Консенсуса, и образуют общее Состояние, доступное по API для Провайдеров и Клиентов. 

Любые действия со смарт-контрактами целевых потребительских программ распространяются и синхронизируются по внутренним протоколам операционной системы между делегатами. Таким образом, у всех делегатов всегда есть одно Состояние кооперативной экономики.

### Клиентское ПО
Провайдеры предоставляют кооперативам клиентское программное обеспечение, в виде мобильных, десктопных, веб и прочих приложений, которые подключаются к операционной системе по API и взаимодействуют с Цифровыми Целевыми Потребительскими программами, отправляя подписанные простой электронной подписью действия и получая информацию из таблиц распределенной базы данных согласно её текущему Состоянию.  



### Сервисное ПО
Провайдеры предоставляют кооперативам сервисное программное обеспечение в виде дополнительных средств автоматизации, которые включают в себя, не ограничиваются:

- средствами оповещения;

- средствами формирования бухгалтерской отчетности;

- средствами авторизации на платформе;

Сервисное ПО исполняется на серверах Провайдеров и обеспечивает полноценное функционирование их клиентского ПО для Кооперативов-Клиентов. 


## Токеномика

### Утилити-токен
Платформа кооперативной экономики использует утилити-токен AXON, свободно передаваемый в рамках контура кооперативной экономики. Токен используется Делегатами для учёта количества переданной, хранимой и обработанной информации Кооперативов в процессе их деятельности. В AXON делегаты получают вознаграждение за предоставленные вычислительные ресурсы. 


### Учётная единица
У каждого кооператива есть внутренняя учётная единица, тикер которой соответствует государственной валюте центрального банка страны регистрации. Например, в России, это RUB. В этой учетной единице ведётся расчет вступительных, паевых и членских взносов согласно законодательству. 


### Эмиссия
Эмиссия AXON производится по протоколу эмиссии, описанному в соответствующем разделе. Эмиссия происходит только тогда, когда экономическая система расширяется, а количество кооперативов-клиентов и спрос на токены от них растёт. 

При уменьшении спроса на AXON, система продолжает функционировать на заданном объеме токенов в обороте до момента начала цикла сжигания. Автоматическое сжигание AXON производится до начала следующего цикла роста спроса на него. Таким образом, система автобалансирует количество токенов в обороте. 

Эмиссия же внутренних учётных единиц кооперативов происходит в процессе приёма паевых взносов, а сжигание - при возврате паевых взносов пайщикам. 


### Оборот
Каждому кооперативу-клиенту для осуществления своей деятельности на платформе необходимо арендовать вычислительные ресурсы у делегатов. Аренда происходит путём преобретия токенов AXON и ежедневной автоматизированной их передаче - делегатам в обмен на квоту вычислительных ресурсов. 

Квота вычислительных ресурсов расходуется по мере увеличения количества завершенных сделок, которые осуществляют пайщики кооператива-клиента. Оплата квоты производится ежедневно с кошелька AXON кооператива. 

Оплаченные AXON перераспределяются в пользу делегатов платформы согласно полученным ими голосам и времени участия. 


## Кооперативные приложения

Ведущие кооперативные приложения Провайдеров могут быть грубо разделены по секторам и типам следующим образом:

- Продукты питания
    - Поставка 
    - Заказ
    - Инвестиции
- Недвижимость
    - Строительство
    - Приобретение 
    - Аренда
- Средства транспорта
    - Поставка
    - Аренда
    - Обслуживание
- Результаты услуг
    - Заказ
    - Поставка

Таким образом, типичные смарт-контракты кооперативной экономики основные экономические отношения через следующие целевые потребительские программы:

- поставка и заказ имущества или результатов услуг;
- аренда имущества;
- инвестиции и спонсорство;

Эти типичные смарт-контракты целевых потребительских программ расширяются до коллективных форм взаимодействия, таких как: коллективные заказы, коллективное пользование, коллективная аренда, коллективное спонсорство и инвестиции, что всё вместе и образует кооперативную экономику как цифровую систему хозяйствования.
